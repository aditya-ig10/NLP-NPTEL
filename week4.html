01 — DECODING

HMM TAGGING
AND VITERBI

TASK

Given words w1...wn, find best hidden tag sequence t1...tn. Naive search is exponential, so we use Dynamic Programming.

delta_j(t+1) = max_i [ delta_i(t) * a_ij * b_j(w_{t+1}) ]
TABLES

delta stores best probability ending in state j at time t. psi stores best predecessor for backtrace.

Final path = argmax at last time step + follow psi backward.

WHY VITERBI

Turns huge combinatorial search into polynomial-time decoding.

USES

POS tagging, speech recognition, bio-sequence labeling.

LIMITS

Unknown words and limited context hurt plain Markov taggers.

02 — PARAMETER ESTIMATION

SUPERVISED
AND BAUM-WELCH

WHEN TAGGED DATA EXISTS
P(tj|ti) = Count(ti->tj)/Count(ti) P(w|t) = Count(t emits w)/Count(t)
MLE

Direct count-based estimation for transition and emission probabilities.

EM LOOP

Initialize A, B, pi -> estimate hidden expectations -> re-estimate parameters -> repeat until convergence.

STATE POSTERIORS
gamma_i(t) = P(X_t=i | Y,theta) zeta_ij(t) = P(X_t=i, X_t+1=j | Y,theta)
FORWARD
alpha_i(t) = P(y1...yt, X_t=i | theta) alpha_j(t+1) = b_j(y_t+1) * sum_i alpha_i(t)a_ij
BACKWARD
beta_i(t) = P(y_t+1...y_T | X_t=i,theta) beta_i(t) = sum_j beta_j(t+1)a_ijb_j(y_t+1)
03 — DISCRIMINATIVE TAGGING

MAXIMUM
ENTROPY

PROBABILITY MODEL
p(y|x) = (1/Z(x)) * exp(sum_i lambda_i f_i(x,y))

Feature functions can include capitalization, suffixes, nearby words, and previous tags.

MAX ENTROPY PRINCIPLE

Among distributions satisfying known constraints, choose maximum entropy. Use only supported evidence; assume nothing extra.

H(p) = - sum_x p(x) log p(x)
Feature: word is capitalized AND tag=NNP
Feature: suffix=ing AND tag=VBG
Feature: previous word=the AND tag=Adj
Feature: prev tag=Det AND current tag=Adj

05 — APPROXIMATE DECODING

BEAM SEARCH

IDEA

At each step, keep only top-k candidate tag sequences. Expand only these, prune the rest. Great speed-accuracy tradeoff.

06 — MODEL COMPARISON

HMM vs MAXENT
vs CRF

HMM

Generative model with transition and emission probabilities.

MAXENT

Discriminative with rich features, but local normalization can cause label bias.

CRF

Globally normalized discriminative sequence model; alleviates label bias.

LABEL BIAS PROBLEM

Locally normalized models can overly prefer states with fewer outgoing transitions. CRFs compare complete paths globally to reduce this bias.
