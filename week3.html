01 — SPARSITY PROBLEM

WHY WE NEED
SMOOTHING

PROBLEM

Unseen but valid n-grams get zero probability with raw MLE.

Zero probabilities break language modeling pipelines because one unseen event can zero out a whole sequence probability.

SOLUTION FAMILY
Good-Turing Estimation
Simple Good-Turing
Absolute Discounting
Kneser-Ney Smoothing
Backoff & Interpolation
GOOD-TURING

Uses frequency-of-frequency counts (Nc) to estimate unseen mass.

ABS DISCOUNT

Subtract fixed d from seen counts and redistribute leftover probability.

KNESER-NEY

Uses continuation probability, not plain unigram, for superior fallback.

02 — CORE METHODS

MODEL
DETAILS

ADJUSTED COUNT
c* = (c + 1) N(c+1) / N(c)

Rare-event counts help allocate probability mass to unseen events. Unseen mass is tied to N1/N.

NOTES

Great for sparse data, but raw Nc values become noisy for larger counts. Simple Good-Turing smooths Nc curve.

FORM
P(wi|h) = (c(h,wi)-d)/c(h) + lambda(h) P(wi)

Typical discount d ≈ 0.75.

IDEA

Reduce every seen count by a constant, then distribute leftover mass via lower-order model.

KEY INSIGHT

Fallback should reward words that appear in many distinct contexts, not just high global frequency.

Pcont(w) ∝ # unique histories before w
WHY STRONG

Kneser-Ney generally outperforms naive unigram fallback for bigrams/trigrams in real corpora.

BACKOFF

Use highest-order n-gram if available; otherwise recursively fall back to lower orders.

INTERPOLATION

Instead of hard fallback, combine trigram + bigram + unigram using lambdas that sum to 1.

04 — COMPUTATIONAL MORPHOLOGY

MORPHEMES,
AFFIXES,
WORD FORMATION

BASICS

Morpheme = smallest meaning-bearing unit. Example: dogs = dog + s, unladylike = un + lady + like.

FREE / BOUND CONTENT / FUNCTIONAL
AFFIX TYPES

Prefix, Suffix, Infix, Circumfix. Also allomorph variation: un-, in-, im-, ir- for similar negation meaning.

INFLECTIONAL

Same lexeme, different grammar: bring, brings, bringing, brought.

DERIVATIONAL

Creates new words: logic → logical → illogical → logician.

PROCESSES

Concatenation, Reduplication, Suppletion, Internal Change.

NLP USES

Lemmatization, morphological analysis, POS tagging, morpheme segmentation, and word generation all depend on morphology knowledge.

05 — FORMAL TOOL

FINITE STATE
AUTOMATON (FSA)

DEFINITION

An FSA is a directed graph with states, labeled transitions, start state, and accept states. It recognizes regular languages and powers morphology pipelines.

WHY NEEDED

Storing every surface word form is infeasible. Rule-driven finite-state processing scales better for rich morphology languages.
