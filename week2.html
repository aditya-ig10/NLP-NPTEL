02 — CORE CONCEPTS

EDIT
OPERATIONS

Four fundamental operations define how we measure the distance between two strings. Click ANIMATE to see each in action.

INSERT
Cost: 1  |  Add a character
cat → cart
DELETE
Cost: 1  |  Remove a character
cart → cat
SUBSTITUTE
Cost: 1 or 2  |  Replace a character
cat → cut
TRANSPOSE
Cost: 1  |  Swap adjacent chars
t e h → t h e
Damerau-Levenshtein only!
TIME COMPLEXITY
O(n·m)
n = length of source string
m = length of target string
SPACE COMPLEXITY
O(n·m)
DP table has n×m cells
ERROR COVERAGE
~80%
of errors within edit distance 1
~100% within edit distance 2

04 — STEP BY STEP

HOW THE
ALGORITHMS
WORK

1
Initialize the DP Table
Set D(i,0) = i for all i (deletions from source). Set D(0,j) = j for all j (insertions to build target). This handles empty string base cases.
2
Iterate Row by Row
For each character X[i] in source and Y[j] in target, compute D(i,j) using three possible previous states.
3
Apply Recurrence
D(i,j) = min of: D(i-1,j)+ins_cost (delete from source), D(i,j-1)+del_cost (insert into source), D(i-1,j-1)+sub (substitute, 0 if match).
4
Read Final Answer
The minimum edit distance is stored at D(n,m) — bottom-right cell of the table.
5
Backtrace for Alignment
Follow pointers (↖ ← ↑) backward from D(n,m) to reconstruct the actual sequence of operations performed.
RECURRENCE FORMULA
D(i,j) = min(
  D(i-1,j) + 1,       // delete
  D(i,j-1) + 1,       // insert
  D(i-1,j-1) + cost   // substitute
                       // cost=0 if X[i]=Y[j]
)
BACKTRACE SYMBOLS
↖ Match / Substitute
← Insertion
↑ Deletion
1
Observe the Input Word x
The user types a word x. It may or may not be correct. We want to find the most probable intended word w.
2
Apply Bayes' Rule
P(w|x) ∝ P(x|w) × P(w). We decompose the problem into: how likely is the typo (error model) and how common is the word (language model).
3
Compute P(w) — Language Model
Estimate word frequency from a large corpus. Common words like "the", "house" get high P(w). Rare words get low P(w).
4
Compute P(x|w) — Error Model
Using confusion matrix, estimate how likely it is that correct word w was typed as x. Nearby keyboard keys, common transpositions get higher probability.
5
Select Best Candidate
argmax_w [ P(x|w) × P(w) ] → return this word as the correction.
THE MODEL
Input word → [Noisy Channel] → x
Original intended word w gets "corrupted"

Find: w* = argmax P(w|x)
Using: P(w|x) ∝ P(x|w) · P(w)

P(w) = language model
P(x|w) = error model
1
Offline: Pre-process Dictionary
For every dictionary word, generate all variants by deleting up to k characters (k=1 or 2). Store word ↔ delete-variants mapping.
2
Online: Process Input Word
When user types a (possibly misspelled) word x, generate all its delete variants (only ~45 for length-9 word at k=2).
3
Hash Lookup
Look up these delete variants in the precomputed hash map. Matches correspond to dictionary words that are within edit distance k.
4
Why Only Deletes?
Insert(a,b) + Delete(b,a) = same word! By only precomputing deletes, we avoid storing millions of insert/substitute variants.
5
Speed Advantage
Naive: compare against entire 100k+ word dictionary. Symmetric Delete: lookup ~45 hash keys. Dramatically faster for real-time applications.
CANDIDATE COUNT COMPARISON
9-char word, edit distance ≤ 2:
❌ NAIVE GENERATE-ALL: ~100,000+ candidates
✅ SYMMETRIC DELETE: ~45 candidates
